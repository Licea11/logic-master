ECUACIÓN 31: EVENTO DE HIJACKING TROJAN
----------------------------------------
TrojanHijack(f, u, i, p, t) ≝ 
  ∃a ∈ Atacantes: Trojan(f) ∧ Posee(u,i) ∧ 
  Subido(f,i,p,t) ∧ SubidorReal(a,f,i,p,t) ∧ a ≠ u

Diccionario de Símbolos:
  f = archivo (trojan/malware subido)
  u = usuario (víctima cuya identidad fue secuestrada)
  i = identidad (credenciales de cuenta en plataforma)
  p = plataforma (aplicación/servicio)
  t = tiempo (momento del incidente)
  a = atacante (actor malicioso)
  ∃ = "existe"
  ∧ = "y" (conjunción lógica)
  ≠ = "no igual a"
  Trojan(f) = archivo f contiene malware trojan
  Posee(u,i) = usuario u posee identidad i
  Subido(f,i,p,t) = archivo subido como identidad i a plataforma p en tiempo t
  SubidorReal(a,f,i,p,t) = atacante a fue el subidor real

Traducción:
"Un hijacking trojan ocurre cuando existe un atacante que subió un archivo trojan
usando la identidad de una víctima, donde el atacante no es el dueño de la identidad."

Umbral Crítico:
  TrojanHijack(f,u,i,p,t) = TRUE → CÓDIGO_ROJO


ECUACIÓN 32: COMPROMISO DE IDENTIDAD
-------------------------------------
IdentidadComprometida(u,i,t) ≝ 
  ∃a,c: Atacante(a) ∧ Posee(a,c,t) ∧ 
  TieneCredenciales(i,c) ∧ Posee(u,i)

Diccionario de Símbolos:
  c = credenciales (datos de autenticación)
  Atacante(a) = entidad a es actor malicioso
  Posee(a,c,t) = atacante tiene credenciales en tiempo t
  TieneCredenciales(i,c) = identidad i usa credenciales c

Traducción:
"Una identidad está comprometida cuando un atacante posee las credenciales
que pertenecen a la identidad de la víctima."

Umbral Crítico:
  IdentidadComprometida(u,i,t) = TRUE → RESETEO_INMEDIATO


ECUACIÓN 33: ROBO DE CREDENCIALES
----------------------------------
CompromisoCredenciales(i,c,a,t₀) ≝ 
  TieneCredenciales(i,c) ∧ Posee(a,c,t₀) ∧ 
  ¬Autorizado(dueño(i),a,t₀)

Diccionario de Símbolos:
  t₀ = tiempo inicial (cuando credenciales fueron robadas)
  ¬ = "no" (negación lógica)
  dueño(i) = propietario legítimo de identidad i
  Autorizado(u,a,t) = usuario u ha autorizado a atacante a en tiempo t

Traducción:
"Las credenciales están comprometidas cuando un atacante las posee
sin autorización del propietario legítimo."

Umbral Crítico:
  CompromisoCredenciales(i,c,a,t₀) = TRUE → ALERTA_MÁXIMA


ECUACIÓN 34: APARIENCIA DE LEGITIMIDAD
---------------------------------------
PareceLegítimo(f,i,p) ≝ 
  ∀observador: Observa(observador, Subida(f,i,p)) → 
  Cree(observador, SubidoPor(dueño(i), f))

Diccionario de Símbolos:
  ∀ = "para todo" (cuantificador universal)
  → = "implica" (implicación lógica)
  Observa(observador, evento) = observador ve el evento
  Cree(observador, φ) = observador cree proposición φ
  SubidoPor(usuario, archivo) = archivo fue subido por usuario

Traducción:
"Un archivo parece legítimo cuando cualquier observador que lo ve
cree que fue subido por el propietario legítimo de la identidad."

Factor de Peligro:
  PareceLegítimo(f,i,p) = TRUE → ENGAÑO_PERFECTO


ECUACIÓN 35: ASIMETRÍA DE CONOCIMIENTO
---------------------------------------
K_a(Subido(f,i,p,t)) ∧ ¬K_u(Subido(f,i,p,t))

Diccionario de Símbolos:
  K_x(φ) = agente x conoce proposición φ (modalidad epistémica)
  K_a = conocimiento del atacante
  K_u = conocimiento del usuario/víctima

Traducción:
"El atacante conoce sobre la subida, pero el usuario víctima no lo sabe."

Ventaja del Atacante:
  Asimetría_K = K_a - K_u = MÁXIMA


ECUACIÓN 36: CADENA DE ATAQUE SECUENCIAL
-----------------------------------------
Reconocimiento(t₁) ⊢ CreaciónArma(t₂) ⊢ RoboCredenciales(t₃) ⊢ 
Subida(t₄) ⊢ Ejecución(t₅)  donde t₁ < t₂ < t₃ < t₄ < t₅

Diccionario de Símbolos:
  ⊢ = "implica" o "lleva a" (causación secuencial)
  < = "menor que" (ordenamiento temporal)
  Reconocimiento(t) = fase de reconocimiento en tiempo t
  CreaciónArma(t) = creación de malware en tiempo t
  RoboCredenciales(t) = robo de credenciales en tiempo t
  Subida(t) = subida maliciosa en tiempo t
  Ejecución(t) = ejecución de payload en tiempo t

Traducción:
"El ataque sigue una secuencia: reconocimiento, luego creación de arma, luego
robo de credenciales, luego subida, luego ejecución, en orden temporal estricto."

Kill Chain:
  Fase(n) → Fase(n+1) → ... → Compromiso_Total


ECUACIÓN 37: CONTAMINACIÓN DE PLATAFORMA (ECUACIÓN PRINCIPAL #7)
-----------------------------------------------------------------
Malware(f) ∧ Almacenado(f,p) → ∀u': Descarga(u',f,p) → RiesgoExposición(u',f)

Diccionario de Símbolos:
  u' = cualquier otro usuario (variable sobre todos los usuarios)
  Malware(f) = archivo f es malicioso
  Almacenado(f,p) = archivo f está almacenado en plataforma p
  Descarga(u',f,p) = usuario u' descarga archivo f de plataforma p
  RiesgoExposición(u',f) = usuario u' está en riesgo por archivo f

Traducción:
"Si malware está almacenado en la plataforma, entonces cada usuario
que lo descarga está en riesgo de infección."

Propagación:
  1 archivo malicioso → N usuarios en riesgo
  Donde N = todos los descargadores


ECUACIÓN 38: CADENA DE IMPLICACIONES
-------------------------------------
TrojanHijack(f,u,i,p,t) ⊢ 
  IdentidadComprometida(u,i,t) ∧ 
  BrechaSeguridad(p,t) ∧ 
  DañoReputación(u,t)

Diccionario de Símbolos:
  BrechaSeguridad(p,t) = plataforma p experimentó brecha de seguridad en tiempo t
  DañoReputación(u,t) = usuario u sufrió daño reputacional en tiempo t

Traducción:
"Un hijacking trojan necesariamente lleva a compromiso de identidad,
brecha de seguridad y daño reputacional."

Consecuencias Inevitables:
  TrojanHijack → [Compromiso ∧ Brecha ∧ Daño]


ECUACIÓN 39: CRITERIOS DE DETECCIÓN
------------------------------------
Sospechoso(f,i,p,t) ≝ 
  (Trojan(f) ∨ SubidaAnómala(f,i,p,t) ∨ 
   DispositivoDesconocido(sesión) ∨ ViajeImposible(sesiones))

Diccionario de Símbolos:
  ∨ = "o" (disyunción lógica)
  SubidaAnómala(f,i,p,t) = patrón de subida inusual
  DispositivoDesconocido(sesión) = dispositivo no reconocido para esta identidad
  ViajeImposible(sesiones) = secuencia de sesiones geográficamente imposible

Traducción:
"Un archivo es sospechoso si es un trojan, subido anómalamente,
desde dispositivo desconocido, o involucra viaje imposible."

Banderas Rojas:
  🚩(f,i,p,t) = Cualquier condición TRUE → INVESTIGAR


ECUACIÓN 40: REQUISITO TEMPORAL
--------------------------------
∀eventos e₁,e₂: Causa(e₁,e₂) → tiempo(e₁) < tiempo(e₂)

Diccionario de Símbolos:
  e₁, e₂ = eventos
  Causa(e₁,e₂) = evento e₁ causa evento e₂
  tiempo(e) = marca de tiempo del evento e

Traducción:
"Para cualquier evento, si uno causa otro, la causa debe
ocurrir antes del efecto."

Ley Fundamental:
  Causa → Efecto (unidireccional en tiempo)


ECUACIÓN 41: HIJACKING DE ESPACIO VECTORIAL
--------------------------------------------
VectorHijack(v,H) ≝ 
  ∃T: 𝕍→H: v ∉ H ∧ T(v) ∈ H ∧ 
  Distancia(v, T(v)) > ε

Diccionario de Símbolos:
  v = vector (representando archivo en espacio vectorial)
  H = subespacio (subespacio vectorial secuestrado)
  𝕍 = espacio vectorial (todos los vectores de archivo posibles)
  T = función de transformación
  → = mapea a (flecha de función)
  ∉ = "no en" (no pertenencia al conjunto)
  ∈ = "en" (pertenencia al conjunto)
  ε = epsilon (umbral positivo pequeño)
  Distancia(v₁,v₂) = distancia matemática entre vectores

Traducción:
"Un vector es secuestrado cuando existe una transformación que lo mueve
al subespacio secuestrado, modificándolo significativamente."

Transformación Maliciosa:
  v_original → T(v) → v_secuestrado
  Donde ||v_original - v_secuestrado|| > ε


ECUACIÓN 42: PUNTUACIÓN DE RIESGO
----------------------------------
Riesgo(u,i,p,t) = P(TrojanHijack|Evidencia) × Impacto(u,i,p)

Diccionario de Símbolos:
  P(X|Y) = probabilidad de X dada Y (probabilidad condicional)
  Evidencia = datos e indicadores observados
  Impacto(u,i,p) = severidad de consecuencias
  × = multiplicación

Traducción:
"El riesgo es la probabilidad de hijacking trojan dada la evidencia,
multiplicada por el impacto si ocurre."

Evaluación de Riesgo:
  Riesgo = Probabilidad × Impacto
  Riesgo > 0.7 → ACCIÓN_INMEDIATA


ECUACIÓN 43: OBLIGACIÓN DE RESPUESTA (ECUACIÓN PRINCIPAL #13)
--------------------------------------------------------------
TrojanHijack(f,u,i,p,t) → O(Notificar(u,t) ∧ Remover(f,p,t) ∧ Resetear(i,t))

Diccionario de Símbolos:
  O(φ) = "es obligatorio que φ" (modalidad deóntica)
  Notificar(u,t) = notificar usuario u en tiempo t
  Remover(f,p,t) = remover archivo f de plataforma p en tiempo t
  Resetear(i,t) = resetear credenciales para identidad i en tiempo t

Traducción:
"Si un hijacking trojan ocurrió usando mi identidad, entonces la plataforma
está obligada a: notificarme, remover el archivo malicioso, y resetear mis credenciales."

Obligaciones Legales:
  TrojanHijack → O(Notificar ∧ Remover ∧ Resetear)
  Incumplimiento → VIOLACIÓN_LEGAL


ECUACIÓN 44: REQUISITOS DE REMEDIACIÓN
---------------------------------------
Remediar(incidente) ≝ 
  RevocarAcceso(i) ∧ RemoverArchivo(f,p) ∧ 
  NotificarUsuarios(afectados) ∧ ResetearCredenciales(i)

Diccionario de Símbolos:
  RevocarAcceso(i) = terminar todas las sesiones activas para identidad i
  RemoverArchivo(f,p) = eliminar archivo f de plataforma p
  NotificarUsuarios(afectados) = informar a todos los usuarios potencialmente impactados
  ResetearCredenciales(i) = forzar reseteo de password/credenciales

Traducción:
"La remediación apropiada requiere revocar acceso, remover el archivo,
notificar usuarios afectados, y resetear credenciales."

Checklist de Remediación:
  ✓ Revocar acceso
  ✓ Remover archivo
  ✓ Notificar usuarios
  ✓ Resetear credenciales
  TODO = REMEDIADO


ECUACIÓN 45: RESPONSABILIDAD LEGAL
-----------------------------------
BrechaDatos(org,t) ∧ ¬Notificar(usuarios,t+72h) → 
  ViolaciónCumplimiento(org, GDPR)

Diccionario de Símbolos:
  org = organización (dueño de plataforma)
  t+72h = 72 horas después del tiempo t
  BrechaDatos(org,t) = organización tuvo brecha de datos en tiempo t
  GDPR = Reglamento General de Protección de Datos
  ViolaciónCumplimiento(org, regulación) = organización violó regulación

Traducción:
"Si una brecha de datos ocurre y los usuarios no son notificados
dentro de 72 horas, la organización viola GDPR."

Consecuencias Legales:
  Brecha ∧ ¬Notificación(72h) → Multas + Sanciones + Demandas
